# ExWHYZ Timeline

ExWHYZの活動記録とファンの思い出をつなぐタイムラインアプリケーション

## 🎯 プロジェクト概要

日本のダンス＆ボーカルグループ「ExWHYZ（エキス）」の輝かしい歴史と、ファンがX（Twitter）で呟いたリアルな思い出をシンクロさせ、熱狂の軌跡をいつまでもエモく振り返ることができる「最強のアーカイブファンサイト」です。

## ✨ 主な機能

### フェーズ1（現在）- v2.4
- ✅ **日付中心のタイムライン表示** - カレンダーの日付を基準に活動情報とTweetを独立管理
- ✅ **2カラムレイアウト** - 左側に活動情報、右側にTweet情報（デスクトップ）
- ✅ **スマホファーストのレスポンシブデザイン** - モバイルでは1カラムに切り替え
- ✅ **ダークモード + ピンクアクセントのエモいUI**
- ✅ **カテゴリ別のイベント分類**（リリース、ライブ、メディア出演等）
- ✅ **2022年6月からの記録** - ExWHYZ活動初期からの完全アーカイブ
- ✅ **Tweetデータの外部ファイル化** - X API連携への準備完了
- ✅ **見出し日付の表示範囲をパラメータで設定可能** - `NEXT_PUBLIC_SURROUNDING_DAYS` で前後日数を指定

### v2.1の主な変更点

#### データの分離
- **活動情報**: `timeline.json`（変更頻度: 低）
- **Tweetデータ**: `user-tweets.json`（変更頻度: 高、将来的にX APIに置き換え）

#### データソースの識別
- `source`: `"static"` または `"x-api"`
- 静的データとAPI取得データを区別

#### 利点
- ✅ データ管理の明確化（活動情報とTweetデータが独立）
- ✅ X API連携への移行容易性（ファイルを置き換えるだけ）
- ✅ 開発・テストの効率化（Tweetデータのみを変更可能）
- ✅ 将来の拡張性（複数のデータソースを統一的に扱える）

## ⚙️ 設定パラメータ

### `NEXT_PUBLIC_SURROUNDING_DAYS`

活動情報の前後何日分の見出し日付を表示するかを指定します。

| 設定値 | 表示される見出し日付の範囲 | 合計表示日数 |
|--------|--------------------------|------------|
| `0`    | 活動日のみ               | 1日        |
| `1`    | 活動日の前後1日          | 3日        |
| `2`    | 活動日の前後2日          | 5日（デフォルト） |

**設定例（`.env.local`）**:
```bash
# 前後1日のみ表示する場合
NEXT_PUBLIC_SURROUNDING_DAYS=1

# 当日のみ表示する場合
NEXT_PUBLIC_SURROUNDING_DAYS=0
```

> **未設定・不正な値の場合**: デフォルト値 `2` が使用されます（後方互換性を維持）
