# ExWHYZ-Timeline 活動情報・ツイートの視覚的差別化 変更要求書（v3.12）

## 📋 変更要件の整理

### v3.12での変更要件

- **活動情報カードの視覚的差別化**:  
  活動情報とツイートのカードが同じ背景色・ボーダーで統一されており、ぱっと見でどちらがどちらか判別しにくい。  
  活動情報カードの背景色を少し明るくして、ツイートカードとの区別をつける。

- **スマホ表示時のツイートカラムへのインデント追加**:  
  モバイル表示（`lg`未満）では活動情報とツイートが同じインデントで縦に並ぶため、  
  どこからどこまでが「その日のツイート」なのか分かりにくい。  
  スマホ表示時のみ、ツイートカラムの右端にインデントを追加し、  
  左側に細いボーダーラインを追加してグルーピングを視覚化する。

---

## 1. 変更の背景と目的

### 現状の課題

| 課題 | 詳細 |
|------|------|
| 活動情報とツイートの背景色が同じ | `EventColumn`（活動情報）と `TweetColumn`（ツイート）のカードがともに `bg-gray-900 border-gray-800` で統一されており、ぱっと見でどちらがどちらか判別しにくい |
| スマホでのグルーピングが不明瞭 | モバイル（`lg`未満）では1カラム表示になり、活動情報とツイートが同じインデントで縦に並ぶため、どこからどこまでが「その日のツイート」なのか視覚的に分かりにくい |

### 変更の目的

- **活動情報を少し目立たせる**: 背景色を少し明るくすることで、活動情報とツイートを直感的に区別できるようにする（洗練された印象を維持）
- **スマホでのツイートグルーピングを明確化**: インデント＋左ボーダーラインにより、ツイートカラムのまとまりを視覚的に示す

## 4. 変更詳細

### 4.1 `components/Timeline/EventColumn.tsx` の変更

#### 変更前（v3.11）

```tsx
export default function EventColumn({ events, categories }: EventColumnProps) {
  // ...
  return (
    <div className="space-y-4">
      <h3 className="text-sm font-semibold text-gray-400 mb-2">活動情報</h3>
      {events.length === 0 ? (
        <div className="bg-gray-900 rounded-lg p-6 border border-gray-800">
          <p className="text-gray-600 text-sm">この日の活動情報はありません</p>
        </div>
      ) : (
        events.map((event) => (
          <div
            key={event.id}
            className="bg-gray-900 rounded-lg p-6 border border-gray-800 hover:border-pink-500 transition-colors"
          >
            {/* ... */}
          </div>
        ))
      )}
    </div>
  );
}
```

#### 変更後（v3.12）

```tsx
export default function EventColumn({ events, categories }: EventColumnProps) {
  // ...
  return (
    <div className="space-y-4">
      <h3 className="text-sm font-semibold text-gray-400 mb-2">活動情報</h3>
      {events.length === 0 ? (
        <div className="bg-gray-800 rounded-lg p-6 border border-gray-700">
          <p className="text-gray-500 text-sm">この日の活動情報はありません</p>
        </div>
      ) : (
        events.map((event) => (
          <div
            key={event.id}
            className="bg-gray-800 rounded-lg p-6 border border-gray-700 hover:border-pink-500 transition-colors"
          >
            {/* ... */}
          </div>
        ))
      )}
    </div>
  );
}
```

**変更点のまとめ:**
- 活動情報カードの背景色: `bg-gray-900` → `bg-gray-800`（1段階明るく）
- 活動情報カードのボーダー: `border-gray-800` → `border-gray-700`（1段階明るく）
- 空状態カードも同様に変更
- 空状態のテキスト色: `text-gray-600` → `text-gray-500`（背景が明るくなるため視認性を維持）
- ホバー時のボーダー（`hover:border-pink-500`）は変更なし

### 4.2 `components/Timeline/TweetColumn.tsx` の変更

#### 変更前（v3.11）

```tsx

export default function TweetColumn({ tweets, loading = false }: TweetColumnProps) {
```

- スマホ表示時（`lg`未満）のみ、左側に細いボーダーライン（`border-l border-gray-700`）を追加
- デスクトップ表示時（`lg`以上）は従来通り（`lg:pl-0 lg:border-l-0`）

## 5. デザイン方針

### 5.1 活動情報の差別化

| 項目 | 変更前 | 変更後 | 意図 |
|------|--------|--------|------|
| 活動情報カード背景 | `bg-gray-900` | `bg-gray-800` | 1段階明るくして目立たせる |
| 活動情報カードボーダー | `border-gray-800` | `border-gray-700` | 背景に合わせて調整 |
| ツイートカード背景 | `bg-gray-900` | `bg-gray-900`（変更なし） | 相対的に暗く見えることで差別化 |

- 大きな色の変化は避け、**1段階の明度差**で洗練された差別化を実現
- ピンクなどの強いアクセントカラーは使わず、グレースケールの範囲内で調整

### 5.2 スマホでのグルーピング表示

| 項目 | 変更前 | 変更後 | 意図 |
|------|--------|--------|------|
| ツイートカラムの左パディング（スマホ） | なし | `pl-4` | 活動情報との視覚的な階層差を作る |
| ツイートカラムの左ボーダー（スマホ） | なし | `border-l border-gray-700` | ツイートのまとまりを細いラインで示す |
| デスクトップ表示 | 変更なし | `lg:pl-0 lg:border-l-0` | 2カラム表示では不要なため無効化 |
